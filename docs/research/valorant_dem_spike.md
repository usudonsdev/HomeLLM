# Research spike: Valorant `.dem` 直接解析

**状態**: ローカル検証中（本番 pipeline 未採用）  
**目的**: 「読めるか／何が取れるか」を先に確認する。できないなら mp4 経路を続ける。

## 前提（現時点の仮説）

| 項目 | 内容 |
|------|------|
| 保存場所 | `%LOCALAPPDATA%\VALORANT\Saved\Demos` |
| 拡張子 | `.dem`（CS2/Dota の Source2 demo と同名だが **別物の可能性が高い**） |
| 公開パーサ | 見つかっていない（Source2 用 parser は Valve 系向け） |
| コミュニティツール | クライアント内視聴のための **ファイル差し替え** が中心（バイナリ解析ではない） |

→ 直接解析は **未確認**。ローカルでヘッダ・構造を見て可否を判断する。

## ローカル試験手順

1. Valorant で試合リプレイをダウンロードする
2. `.dem` をリポ外または `.local/research/dem/` にコピー（Git に載せない）
3. プローブを実行:

```powershell
powershell -File .\scripts\research\probe-valorant-dem.ps1 -Path "$env:LOCALAPPDATA\VALORANT\Saved\Demos\<file>.dem"
```

4. 結果を見る観点:
   - 先頭マジック／ASCII 文字列があるか
   - zlib/snappy らしき圧縮ブロックがあるか
   - Source2 demo パーサで読めるか（**読めなければ別フォーマット確定に近い**）
   - ラウンド境界・キル・マップ名などが機械可読で出るか

## 判定基準

| 結果 | 次の一手 |
|------|----------|
| 構造化イベントが取れる | `valorant-dem` pipeline を ADR 化して実装検討 |
| クライアント専用バイナリのみ | mp4 / 画面録画経路を継続。`.dem` は入力候補から外すか「メタデータだけ」に限定 |
| 部分的に読める | 取れる範囲（例: マップ ID、長さ）だけ ADR に書いて小さく採用 |

## セキュリティ

- `.dem` は個人の試合データ。リポジトリ・公開 Issue に生ファイルを載せない
- ToS 上、リプレイ共有・改造クライアントはリスクがある。解析スパイクは **自分のファイル・オフライン** に限る
